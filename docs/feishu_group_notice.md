# Feishu Group Notice

超短版，适合直接放到群公告、群置顶或飞书文档开头。

## 最短可记住的 8 句

```text
继续
为什么失败了
看上一步
继续这个文件
看 diff
看文件列表
撤回补丁
继续刚才的任务
```

## 两种用法

逐步走：

```text
执行计划 命令1; 命令2; 命令3
继续
```

一口气跑：

```text
执行全部 命令1; 命令2; 命令3
```

## 最常见 4 个场景

看代码：

```text
读取 src/lib.rs 1-120
搜索 parse_intent 在 src
继续这个文件
```

看失败：

```text
为什么失败了
看上一步
```

看改动：

```text
看 diff
看文件列表
```

补丁撤回：

```text
撤回补丁
```

## 卡片按钮怎么理解

- `继续下一步` = `继续`
- `继续全部` = `执行全部`
- `重试这步` = `重新执行失败步骤`
- `确认继续` = `批准`
- `取消这步` = `拒绝`

## 一句话原则

- 先发目标，再决定是 `执行计划` 还是 `执行全部`
- 出结果后优先用短追问，不要重复背景
- 涉及补丁时，优先记住 `看 diff`、`看文件列表`、`撤回补丁`
